# Solutions Architect Persona

## Role
Solutions Architect. Designs end-to-end technical solutions that connect customer requirements to product capabilities. Owns reference architectures, integration patterns, and proof-of-concept designs. Lives at the intersection of pre-sales engineering and hands-on technical design.

## Priorities
- Integration points with existing customer infrastructure
- Reference architectures and proven deployment patterns
- Production readiness and day-two operations
- Scalability characteristics and performance at target load
- Migration paths from current state to proposed solution
- High availability, disaster recovery, and data protection
- Multi-cloud and hybrid deployment options
- Proof-of-concept scoping and success criteria

## Reading Patterns
- Looks for architecture diagrams and integration patterns first.
- Evaluates how this fits into a typical enterprise stack.
- Checks for production deployment examples at realistic scale.
- Searches for migration guides and coexistence strategies with legacy systems.
- Reviews API documentation and integration mechanisms in detail.

## Common Concerns
- "How does this integrate with the systems my customer already runs?"
- "Is there a reference architecture I can use as a starting point?"
- "What does a production deployment look like at 1,000 users? At 100,000?"
- "What is the migration path from their current solution?"
- "How does this handle high availability and disaster recovery?"
- "Can I deploy this on-premises, in the cloud, or both?"
- "What are the hard dependencies and prerequisites?"
- "What does the proof-of-concept look like, and how long does it take?"

## What Makes Them Stop Reading
- No architecture diagrams or integration details
- Missing production deployment guidance
- Only covers greenfield scenarios with no migration story
- No mention of scalability limits or performance characteristics
- Ignoring hybrid and multi-cloud requirements

## What Wins Them Over
- Clear reference architectures with realistic component diagrams
- Documented integration patterns for common enterprise systems
- Production case studies with scale and performance data
- Detailed migration guides with phased rollout plans
- Honest discussion of limitations, prerequisites, and what this does not solve
